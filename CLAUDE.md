# taxprosto.pro

## Продукт
Вебзастосунок для самостійного заповнення та подачі річної податкової декларації про майновий стан і доходи (форма **F0100215**) громадянами України.

**Ключовий flow:**
1. Користувач завантажує довідку про доходи — форму **F1419104** «Відомості з ДРФО» (PDF з Електронного кабінету / Дії)
2. Застосунок парсить дані (податковий агент, нарахований дохід, утриманий ПДФО/ВЗ за кодом 101)
3. Формує валідний XML-файл за XSD-схемою ДПС (F0100215)
4. Користувач завантажує XML → імпортує в Електронний кабінет (cabinet.tax.gov.ua)

## Аудиторія
Фізичні особи — платники податків в Україні, що подають річну декларацію самостійно: ФОП 3 група, фізособи з іноземними доходами, ті що мають право на податкову знижку.

## Стадія
MVP працює end-to-end: парсинг F1419104 → перевірка → генерація XML → завантаження. XML **валідовано за офіційною XSD ДПС**. Задеплоєно live на GitHub Pages.

Поточне покриття: **зарплата код 101** з авто-маршрутом **10.1** (ПДФО 18%) / **10.3** (Дія Сіті, ПДФО 5%) + **код 129 → рядок 11.3** (не оподатк.). Декларація за **2025 рік**. Самостійні зобов'язання (гр.6, додатки Ф1/Ф2) — поза MVP.

**Домен**: https://taxprosto.pro/ (live, HTTPS) — деплой з репо `github.com/VOhnivets/taxprosto-pro`, гілка `deploy-clean`.

Відкриті питання MVP: Дія Сіті (10.1 vs 10.3 при ПДФО 5%), C_REG/C_RAJ та адреса (немає в довідці), інші типи доходів (код 129 тощо), тестовий імпорт у cabinet.tax.gov.ua.

## Технічний контекст

- **Схема XML**: форма **F0100215** (наказ МФУ №859, у ред. наказу №119 від 26.02.2025). Локально: `knowledge/xsd/F0100215.xsd` + `common_types.xsd`. Фіксовані коди: C_DOC=F01, C_DOC_SUB=002, C_DOC_VER=15.
- **Джерело даних**: довідка про доходи з **Дії** — це форма **F1419104** «Відомості з ДРФО» (PDF). Саме її видає Дія за запитом. Формат: таблиця доходів за місяцями (податковий агент, нарах./випл. дохід, ПДФО, ВЗ, код ознаки доходу).
- **Цільова система**: cabinet.tax.gov.ua → «Звітність» → «Імпорт XML»
- **Валідація**: `xmllint --schema knowledge/xsd/F0100215.xsd` (схеми у windows-1251). XML генерується у windows-1251 (вимога ДПС).
- **Архітектура**: 100% client-side — застосунок `index.html`, без серверів і збереження даних. PDF.js (CDN) для парсингу.

## Структура папок

- `index.html` — увесь застосунок (client-side)
- `knowledge/xsd/` — офіційні XSD-схеми ДПС (reference)
- `knowledge/rules/` — фото форми декларації (gitignored, персональні)
- `output/` — прототипи, one-pager
- `.claude/plans/` — плани реалізації фіч
- `.claude/skills/tax-consultant/` — skill податкового консультанта для підтримки розробки
